🤖 TRADING INTEGRATION — Phased Roadmap
Goal: evolve Meridian from a research terminal into a semi-automated and eventually fully automated trading platform. Built in phases — paper trading before any real money. Broker: Alpaca (free API, commission-free, built-in paper trading environment).
Phase 1 — Live Prices
Already on roadmap · Pre-requisite
Auto-fetch current prices for all tickers via Polygon.io or Yahoo Finance API. Populate Current Price + % change from entry in Stock Tracker tab. Foundation everything else is built on.
Phase 2 — Price Alerts
New Tab — Alerts
Set price targets and stop levels for any tracked ticker. Meridian checks prices on refresh and fires a browser notification when a level is hit. You still act manually — this is the notification layer before any automation. No broker connection needed at this stage.
Phase 3 — Quick Order
Stock Tracker integration
Add a "Trade" button next to each ticker. Clicking it generates a pre-filled order form — ticker, direction (buy/sell), quantity, order type (market/limit). You review all details, then one click opens your broker's trade ticket with parameters pre-populated. You confirm and submit manually. No API key needed — just a deep-link to your broker.
Phase 4 — Paper Trading
Alpaca API · Simulated only
Connect Alpaca's paper trading API (free, no real money). Execute simulated trades directly from Meridian to test strategies. Track paper P&L alongside your research. Validate your AI-assisted research edge before risking real capital. Alpaca gives $100K simulated buying power by default. Requires Alpaca account + API key stored locally.
Phase 5 — Agent Triggers
Claude-powered · Semi-auto
Claude monitors the news feed and price alerts, evaluates trigger conditions against your research thesis, and recommends trades with reasoning. You approve or reject each recommendation before execution. Claude is the analyst layer — you remain the decision maker. Connects news feed → analysis → suggested order → your approval → Alpaca execution.
Phase 6 — Full Automation
Long-term · Real money
Fully autonomous agent executes trades against pre-defined rules: position sizing, stop losses, max exposure per sector, daily loss limits. Real money via Alpaca live API. Extensive paper trading validation required before activation. Kill switch always accessible. Regulatory note: automated trading must comply with pattern day trader rules (PDT) — $25K minimum account for unlimited day trades.
🗄️ DATABASE INTEGRATION — Live Data Management
Goal: replace hardcoded HTML data with a live database so you can add, edit, and remove tickers, unicorns, and watchlist entries directly from Meridian — no coding required. Recommended stack: Google Sheets as the database (free, familiar, no setup) → Supabase for the full production upgrade when ready.

Free · No setup required
Your Google Sheet becomes the database. One sheet per data type: Tickers, Unicorns, IPO Calendar, Political Holdings. Meridian fetches all data on load via Google Sheets API and renders it dynamically. Adding a ticker = adding a row to the sheet. Instant, familiar, zero cost. Requires a Google account and a published Sheet (read-only public link).
Stage 2 — Inline Add/Edit
Meridian UI forms
Add a "+ Add Ticker" button to the Stock Tracker tab. A modal form appears — fill in ticker, company, sector, thesis, risk factors. Hit Save and it writes to your Google Sheet via the Sheets API and updates the table in real time. Same for unicorns, IPO entries, and political holdings. No sheet-opening required — manage everything from Meridian directly.
Stage 3 — Edit & Delete
Full CRUD operations
Every row in every table gets ✏️ Edit and 🗑️ Delete buttons. Click Edit — a pre-filled modal opens with all the current data. Make changes, hit Save — updates Google Sheet and re-renders the row instantly. Click Delete — confirmation prompt, then removes the row from the Sheet and the table. Full create/read/update/delete (CRUD) without ever touching a spreadsheet or CSV.
Stage 4 — Supabase Upgrade
Production database · Free tier
When Meridian moves to GitHub Pages and needs a proper backend, migrate from Google Sheets to Supabase — a free, open-source PostgreSQL database with a REST API. Faster queries, better security, supports user authentication (so only you can write data), real-time subscriptions (data updates across all open browser tabs instantly), and scales to millions of rows. Free tier: 500MB storage, unlimited API calls.
Stage 5 — Claude Auto-Populate
AI-assisted data entry
Type a ticker symbol into Meridian. Claude looks it up automatically — pulls current price, market cap, revenue, key investors, thesis summary — and pre-fills the Add Ticker form. You review, adjust if needed, and hit Save. Research that currently takes a full session gets compressed into seconds. The database becomes self-populating with AI assistance.
LONG-TERM VISION: Meridian evolves from a static research file into a live, AI-powered trading terminal — auto-updating prices, embedded charts, news feed, bubble chart analytics, a live database you manage entirely from the browser, a Claude agent that monitors your watchlist and executes trades within rules you define, and a password-protected admin panel with an embedded AI analyst that knows your entire portfolio. 💰 COMMERCIALIZATION — Legal, Licensing & Monetization
Goal: legally charge subscription fees for Meridian as a commercial product. Key insight: as a pure research/data tool (not personalized investment advice) the regulatory barrier is low. The moment trade execution is added, structure it so users connect their own brokerage accounts — keeping Meridian in the "tools" category, not "broker-dealer" territory.
Step 1 — Business Formation
~$100-500 · LLC or Corp
Form an LLC (or Delaware/Wyoming Corp for investor-friendliness). Required for liability protection, payment processing, and business banking. Services: Stripe Atlas, LegalZoom, or your state's Secretary of State website directly. This is the foundation everything else is built on.
Step 2 — Legal Documents
~$500-2,000 or DIY
Three required documents: (1) Terms of Service with clear investment disclaimer — "information only, not personalized investment advice." (2) Privacy Policy — required by law if collecting any user data. (3) Subscription and refund policy. Draft with AI assistance, have a startup attorney review. One-hour consultation ~$300-500.
Step 3 — Data Licensing
Key blocker for commercial launch
Yahoo Finance and TradingView are free for personal use — commercial redistribution requires licensing. Switch to: Polygon.io ($29/mo — explicitly allows commercial use, real-time data) for price data. Intrinio or Tiingo as alternatives. News RSS feeds: most allow commercial use but verify per source. This upgrade also solves the 15-minute delay issue.
Step 4 — Payment Processing
Stripe · One session to build
Stripe handles subscriptions, tax collection, failed payment retries, invoicing, and international payments. Integrates with Supabase Auth for subscription gating — users who pay get access, users who don't are blocked. Suggested pricing tiers: Free (read-only), Pro $20-29/mo (full features), Team $49-99/mo (shared chat + multi-user). Stripe takes 2.9% + 30¢ per transaction.
Step 5 — User Auth + Gating
Supabase Auth · Already on roadmap
Supabase Auth (already in DB roadmap) handles user accounts, login, and session management. Stripe webhook updates Supabase user record when payment succeeds or fails. Meridian checks subscription status on load — Pro features locked for free users. Replaces the current localStorage password with proper per-user authentication.
Step 6 — Regulatory Boundaries
What NOT to do
✓ OK — no registration needed: Data/research tools, market data display, educational content, portfolio tracking, AI-assisted research (framed as information not advice).
⚠ Requires structure: Trade execution — use user-owned Alpaca accounts. Meridian sends signals to THEIR account, never touches money directly.
✗ Requires RIA registration: Personalized investment advice, managing money on behalf of clients, discretionary trading of client funds. Avoid these entirely.
Step 7 — The Competitive Moat
Why this is a real product
Bloomberg charges $24K/year and has no AI layer. TradingView has charts but no portfolio-aware Claude assistant. Finviz has screeners but no unicorn tracking or political holdings intelligence. Meridian's moat is the combination: AI research + curated intelligence layer + trade execution + real-time data — at $20-50/month. That's a real gap in the market.

Options Flow Tab
Unusual Whales API · $50/mo
Real-time feed of unusual options activity. Ticker, expiry, strike, call/put, premium, side, exchange. Color coded: green sweeps (aggressive buyers hitting multiple exchanges simultaneously — strongest bullish signal), red puts, yellow dark pool. Filter by ticker, min premium ($100K+), expiry. Auto-flags activity on your tracked tickers. Sweep orders are the key signal — they indicate institutional conviction.
Dark Pool Monitor
Unusual Whales API · Same subscription
Large off-exchange block trades. When $50M+ of a tracked ticker trades in a dark pool, it surfaces immediately. Dark pool prints often precede major price moves — institutions executing large positions before news. Cross-reference with our political holdings tab for maximum signal.

For every crypto in your tracker (BTC, ETH, SOL etc): exchange inflow/outflow trends, holder concentration (top 10 wallets % of supply), DEX vs CEX volume ratio, active addresses trend.
